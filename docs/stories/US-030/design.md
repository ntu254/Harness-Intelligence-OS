# Design

## Domain Model

US-030 implements durable capture for the US-029 `friction-event` contract.

Canonical fields:

- `event_id`: UUID-like event identity generated by the CLI unless supplied.
- `schema_version`: `1.0.0`.
- `artifact_type`: `friction-event`.
- `friction_type`: one US-029 taxonomy type.
- `severity`: `low`, `medium`, or `high`.
- `source`: `trace`, `agent`, `review`, `workflow`, or `provider`.
- `summary`: human-readable explanation.
- `observed_at`: RFC 3339 timestamp.
- optional linkage: `story_id`, `trace_id`.
- optional context: `provider`, `affected_paths`, `evidence`,
  `proposed_action`, `notes`.

## Interface

```text
harness-cli friction add \
  --type provider_unavailable \
  --severity high \
  --source trace \
  --summary "NotebookLM profile was unavailable during proof capture." \
  --story US-026 \
  --trace 24 \
  --provider notebooklm-mcp-cli \
  --evidence-command "nlm login --check" \
  --evidence-exit-code 1 \
  --evidence-details "Profile not found: default"

harness-cli query friction-events
```

## Validation Semantics

- Unknown friction type fails.
- Unknown severity fails.
- Unknown source fails.
- `provider_unavailable` requires `--provider`.
- `high` severity requires evidence.
- `observed_at`, if supplied, must look like RFC 3339.
- Story and trace foreign keys must exist when supplied.
- Capturing friction does not close backlog items or change story proof flags.

## Data Model

Migration `007-friction-events.sql` adds a `friction_event` table. Structured
events are queryable and story-linked events render in context packs. The table
stores arrays/objects as JSON text to avoid adding suggestion-specific schema
before US-031/US-032.

## Guardrails

- No automatic policy mutation.
- No backlog suggestion engine.
- No rule improvement proposal engine.
- Free-text trace friction remains supported.
